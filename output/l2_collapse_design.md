## 步 1｜機制坐實

### 1.1 合併鍵＝雙鍵 union-find 傳遞閉包（pipeline.py:248-257）

```python
for b in bindings:
    union(b["id"], ("sw", b["swatch"]["page"], tuple(b["swatch"]["bbox"])))  # 同色樣實例
    union(b["id"], ("code", b["code"]))                                       # 同碼
```

每條綁定同時 union 到「它的色樣實例節點」與「它的碼節點」。兩者皆為 union 鍵 ⇒
**傳遞閉包**：只要圖上連通就併成一個 cluster，一 cluster = 一 Variant。設計原意
（D_DESIGN §D1）＝同碼跨頁合併（合法）＋同色樣實例內多碼同色（合法）。病灶＝
這兩條合法邊在特定結構下連鎖爆炸。

### 1.2 兩個放大器把「合法邊」連鎖成巨團

- **Hub 色樣**：一個大面積色樣（情境照、馬賽克示意版）被塊綁（`ok_blk`,
  pipeline.py:366）掛上**多個碼** → `("sw", 該色樣)` 鍵把這些碼的綁定全 union 在
  一起。
- **橋接碼**：同一個碼出現在**多個色樣實例**（跨頁）→ `("code", 該碼)` 鍵把這些
  色樣的 cluster 全 union 在一起。A02G 最大團含 **32 個橋接碼**（各跨 ≥2 色樣）。

兩者交織：hub 把「碼」聚在一起、橋接碼把「色樣」聚在一起，交替傳遞 ⇒ 整檔連通。

### 1.3 med_ex 救回為何觸發 A02G 塌縮

v13 med_ex 降中位 → icon B 臂門檻降 → A02G p37-40 少殺、**46 碼由佇列升為塊綁
綁定**（v13_FREEZE 已載：dm 46→0、blk +46）。這 46 條新綁定＝union 圖上 **46 條
新邊**，把原本 v12 下分離的多個 cluster 接通過臨界 → v12 最大團僅 18 綁定、v13
暴增為 **119 綁定/23 頁**。救回本身正確（scan 層親驗通過），塌縮是**下游組裝層**
把救回碼餵進無守衛的 union-find 的後果，非救回錯誤。

### 1.4 color=None 由來（塌縮的可見簽名）

巨團含多個色樣、色名互相衝突（A02G：`LUME…`/`CROGIOLO…`/`None`…30+ 名；
Topcer：`HEXAGON`/`TRIANGLE`/`LOZENGE`…23 名）→ `assemble()` 判 `len(colors)>1`
→ `color_en=None` + `code_color_conflict` 佇列項（pipeline.py:294-301）。**故塌縮
Variant 的色名衝突「有」被佇列部分揭露，但「23 頁誤併成一團」的結構量級完全不可見。**

### 1.5 SL-6~8（A102 L2 加固）為何沒接住＝覆蓋缺口，非觸發序

SL-6~8 的 `merge_key_suspect`（pipeline.py:219-232）條件 C3 首要前提 **P＝碼首字母
∈ band_letters**。P 是承重設計（s22ext §四）：無 P 的 C0 型在 dev 誤降 779 筆
合法綁定，故刻意窄化到「有 routed band 家族的字母碼」。

A02G/Topcer/Provenza 的橋接碼全是普通產品碼/馬賽克碼（`M6RN`/`EJ7R`/純數字
`16`…），**首字母無 band 家族 → P 不成立 → C3 一律不觸發**。實測：A02G 32 橋接碼
band 開頭 **0/32**、Provenza 8/8 全 0、Topcer 23/23 全 0。**結論：條件不涵蓋
（scope gap），不是觸發序問題。**

> 復現：`python3 dev/assembly_probe.py catalogs --detail catalog_A02G-04_en`

### 1.6 與 A102 嵌合體同根論證

| | A102 嵌合體（v5/v6 期） | 本次 L2 塌縮（A02G/Topcer…） |
|---|---|---|
| 層 | D 組裝 union-find 合併鍵 | **同層** D 組裝 union-find 合併鍵 |
| 病灶 | 一個值錯誤碼（band 家族）跨多色樣 → 誤當身分鍵 → 嵌合 | 橋接碼/hub 色樣跨多色樣頁 → 誤當身分鍵 → 巨團塌縮 |
| 共同根 | **合併鍵無條件信任「共享一個 code/swatch＝同一產品」** | **同一根** |
| 差異 | 訊號＝band 家族（少數色樣、單頁列向） | 訊號＝普通碼/馬賽克碼（多色樣、跨頁）、量級大 10~100× |
| A102 補丁下場 | SL-6~8 以 **P（band 字母）** 窄化簽名接住 | **P 不涵蓋非 band 碼 → 同病復發** |

**同一病灶、同一層第二次漏網。** A102 的補丁是「簽名特化的窄補」（band 字母 +
橋接），非結構性守衛；一旦橋接碼不是 band 字母，同根病灶原樣復發。這正是制度補洞
條款的溯源（§4）：**逐案打補丁治不了「合併鍵無條件信任身分」這個結構病，需要
結構性守衛 + 組裝層守恆儀器。**

## 步 3｜修法設計（per-doc、失敗方向＝寧不合併，不動 core，設計送審）

### 3.1 塌縮乾淨簽名（cluster 層後驗，全語料 0 誤旗）

union-find 完成後，一個 cluster 同時滿足：
1. **跨頁**：其 `mergedFrom` 涵蓋 **≥2 個不同頁**；
2. **色名衝突**：解析出 `code_color_conflict`（`len(colors)>1`，即 color=None 因
   衝突而非因無色）；
3. **量級**：`mergedFrom ≥ N`（塌縮量門檻）。

實測：此三合一簽名在 catalogs〜catalogs7 全 82 檔命中**恰 7 檔塌縮、0 健康檔**
（Ariostea 單頁 82 被條件 1 排除；合法同碼跨頁合併＝色名一致 → 被條件 2 排除）。
純數學論證失敗方向：**同一產品跨頁合併不會有衝突色名**；衝突色名 + 跨頁＝被併的
色樣多半是不同產品被錯鏈——故此簽名的失敗方向＝漏旗（塌縮沒被逮），**絕不會
誤旗合法合併**（合法合併無衝突）＝符合鐵律 2 失敗方向。

### 3.2 守衛動作（失敗方向＝寧不合併，不自信錯綁）

命中簽名的 cluster **不得輸出為單一 Variant**。兩個候選，建議採 (A)：

- **(A) 降級佇列（建議，最保守）**：整團綁定逐筆落佇列
  `reason=assembly_collapse_suspect`（帶 prov+swatch，資料無損），**不出任何
  Variant**。符合鐵律 2「不確定一律進佇列、絕不自信錯綁」＝把 119 頁跨頁誤併留給
  人工/VLM，而非出一個自信錯的巨 Variant。守恆式更新：
  `Σ mergedFrom + assembly_collapse_demoted = 凍結綁定總數`（同 A102 降級帳法）。
- **(B) 退回逐色樣實例（次選）**：斷開跨頁鏈、按 `(page,bbox)` 分組各出一 Variant。
  保留較多 Variant（呼應票面「留多個 Variant 待人工」），但 hub 色樣（單色樣多碼）
  仍會出一個頁內 color=None 團——**未完全消除自信錯綁**，故列次選、送審裁。

> 建議 (A)：本專案鐵律 2 高於「多留 Variant」，且塌縮團的每筆綁定都有 prov+swatch，
> 佇列化零資料損失、且天然接 D 板 VLM 承接（拼磚頁本就待 VLM）。

### 3.3 與 A102 SL-6~8 併 case（不重複造補丁）

兩防線**互補、不衝突**，同層各司一職：

- **SL-6~8（既有，早攔／訊號級）**：band 字母橋接碼在 union **之前**降級——L1 訊號
  漏網時的獨立防線，處理「值錯誤 band 碼」特化型，已凍結、逐位不動。
- **本案守衛（新增，後驗／結構級）**：union **之後**在 cluster 層逮塌縮結果——
  **不依賴碼字母**，覆蓋 SL-6~8 的非 band 缺口（A02G/Topcer/Provenza 全部）。

論證是否收編 SL-6~8：後驗守衛在結構上**涵蓋** SL-6~8 攔的所有嵌合（band 嵌合亦是
色名衝突 + 跨色樣）。但收編需重跑 SL-6~8 的 779 誤降分離量測 + c7 A102 夾具複跑，
確認後驗簽名對 A102 型「值錯誤但單頁列向」也正確處置——**列為實作前第一優先量測
（通案二），本設計案不預先收編、SL-6~8 維持凍結。**

### 3.4 常數與校準（實作前必辦，通案二/鐵律 4）

- **N（塌縮量門檻）＝唯一候選新常數**，須宣告 + 校準 + 凍結。校準材料：塌縮病例
  下界（v13 誘發最小＝Uniche 27；產線既存最小＝Provenza 30）vs 合法跨頁合併上界
  （需補量：dev 全體「色名一致的跨頁合併團」最大綁定數分布）。**本設計案不預設
  N 數值**——留給實作前分離量測定，落 FREEZE。（初步：合法合併團色名一致，天然
  與塌縮團＝衝突色名分離，N 可能只需防「2 筆色名 typo」的下限保護。）
- 失敗方向自檢：條件 2（色名衝突）已使誤殺合法合併＝結構不可能；N 只調「多小的
  衝突團才值得逮」，動 N 不會誤傷合法合併＝安全方向。
