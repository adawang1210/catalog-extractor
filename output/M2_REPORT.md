## 5. 人工 GT(6 頁抽樣;裁決 v2 留之後執行)

抽樣:Emil p15 / Provenza p16 / Topcer p5 / Level p216 / Ariostea p36 / Viva p14(選頁規則見

### 三個數字(以 v2=定案版為準;v1 對照)

| 指標 | v2 | v1 |
|---|---|---|
| **規則 1 y 帶搶綁錯綁率** | **80/266 = 30.1%**(越頁 38=14.3%、同頁家具 42=15.8%) | 同 80(共同錯) |
| **far precision(far 中綁對比例)** | **29/31 = 93.5%**(0 筆真錯綁在佇列中) | 1/13 = 7.7% |
| **真實正確率** | **186/266 = 69.9%**(proxy 90.4%,高估 20.5pt) | 135/266 = 50.8%(proxy 96.0%,高估 45.2pt) |

### 解讀

- **v2 殘餘錯誤 100% = 規則 1**(80/80 皆 d=0 搶綁):x 親和性已把規則 3 錯誤清零
  (v1 另有 47 筆規則 3 越頁錯 + 4 筆同頁錯,v2 全修)。v2 較 v1 真實正確率 **+19.1pt**——
  與 proxy(-0.9pt)方向相反,證實 proxy 失真。
- **規則 1 搶綁的兩型**:(a) 越頁——spread 頁代碼落在對頁色樣 y 帶(Emil Angolare 10 筆→對頁
  木紋板、Provenza 25 筆→對頁馬賽克、Viva 3 筆→對頁認證 logo);(b) 同頁家具——高長條圖形
  y 帶罩住整區(Topcer 22 筆→「VICTORIAN BORDERS」側欄、Ariostea 20 筆→表格分隔條)。
  共同病因:**規則 1 無 x 約束 + 色樣抽取收進頁面家具**。下一輪頭號目標。
- **far 佇列語意在 v2 下反轉**:v1 的 far≈真錯(92%);v2 的 far≈「綁對但遠」(93.5% 正確、
  0 真錯)。真錯誤(規則 1)d=0 永不觸發 far → **far 作為錯誤偵測器對現存錯誤 recall=0**,
  仲裁預算應改投「無 x 對齊色樣可綁的代碼」而非 far 佇列。
- **候選純度**:57/323(17.6%)是 junk token(價格帶 A\d+ ×35、厚度標記 CM2 ×20、尺寸 ×2)。
  另見偵測缺口:>8 字元 SKU(Ariostea P612562S8/UCC6S300562)與全字母 SKU(Viva EGUC、
  Level ELMS)不進候選——Stage 2 問題,綁定指標未涵蓋。
- 正面確認:標準頁型近乎全對——Level p216 30/30、Topcer 主體 69/69、Emil/Provenza 左半
  圖說區 34/34。錯誤高度集中於矩陣/規格表/家具頁型,非隨機。
