# §0.2 綁定 Spike 報告 — 純幾何基線(2026-07-08)

## 結論(一句話)
**純幾何綁定在四種代表頁型上拿到 50/50(100%),零模型、零 API、零成本;建議 Stage 3 主力改為「幾何綁定 + 文字層驗證」,VLM 降級為低信心頁的仲裁者(免費層即可)。**

## 方法
`spike_geom.py`:色樣抽取(XObject bbox,面積 60pt²~55% 頁面;無 XObject 時退 `get_drawings()` 向量填色矩形)+ **反向 token 指派**——每個文字 word 依序套三條內容推導規則指派給色樣:
1. **同列**(y 帶重疊 → 給 x 最近的色樣)
2. **圖說**(x 重疊且在色樣下方不遠 → 給上方最近的色樣)
3. **垂直區段**(給起點在其上方、最近的色樣)

無任何 per-vendor / per-版型寫死;GT 由人工目檢 render 圖標註。

N:M 案例驗證:`MQ7Z` 被 p3 左右兩張照片同時引用,兩邊都正確捕捉。

## 過程中抓到的坑(比分數更有價值)
1. **輸出截斷造成假失敗**:多列區段第 2、3 列代碼被 `[:300]` 截掉(Marazzi 一度 67%)。
2. **抽取漏色樣會造成假成功**:大情境照被面積上限濾掉後,孤兒圖說全倒給倖存的圖,scorer 的「唯一命中」驗不出來(p3 一度虛報 5/5)。→ **production 必加「抽取覆蓋率檢查」:有代碼圖說但附近無色樣 → 標 needs_review**。
3. Topcer 的「無代碼」是 census 排除純數字 token 的誤判;色 chip 面積門檻也一度濾掉全部 30 個 chip。門檻改為絕對下限 60pt²。

## 誠實的限制
- n=4 頁,且三條規則是看著這些頁設計的——**有 spike 級過擬合風險**,M1 要在 13 份 corpus 的更多頁型上驗(尤其:跨欄雜誌版型、旋轉頁、雙欄表格)。
- Scorer 驗「code 唯一命中 + 色名同在」,不是像素級 swatch 身分;已用目檢補驗四頁的分群正確性。
- 抽取仍會撈進非色樣圖(Topcer 頂部尺寸樣、頁首雜訊 token)——這是 Stage 2/7 分類的工作,不影響綁定正確性但影響輸出乾淨度。

## 升級路徑評估(依使用者限制:無付費 key、無 GPU、無標註人力)
| 選項 | 可行性 | 建議 |
|---|---|---|
| **純幾何(本報告)** | ✅ 零成本、本機、快 | **主力**。配 Stage 4 token 驗證 + 覆蓋率檢查 |
| 免費託管 VLM(Gemini AI Studio 免費層) | ✅ 需使用者開免費 key;有 RPM/RPD 限額 | **仲裁者**:只送低信心/驗證失敗頁 + gold set 預標交叉檢查。限額對這用量夠 |
| 本地小 VLM(SmolVLM/Moondream/Florence-2,西方來源) | ⚠️ Apple Silicon 可跑(MLX/Metal),模型 2~5GB 可放 USB 碟;但小 VLM 對密集網格的空間綁定普遍弱 | **暫不做**。幾何殘差如果集中在特定頁型再回頭評 |
| 付費 VLM API | — | 目前無必要 |
